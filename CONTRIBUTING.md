# Contributing to SPV-FINDER

Thank you for your interest in contributing! We welcome all types of contributions, including bug fixes, new features, documentation improvements, and feedback.

Because our `main` branch is fully protected to ensure code stability, please follow the structured contribution workflow below.

## 🚀 Getting Started

### 1. Fork and Clone
Do not try to push directly to this repository. Instead, fork the project first:
1. Click the **Fork** button at the top right of this repository page.
2. Clone your personal fork to your local machine:
   ```bash
   git clone https://github.com/Luche/spv-finder.git
   cd spv-finder
   ```

### 2. Create a Feature Branch
Always create a isolated branch before making changes. Never work directly on your `main` branch.
```bash
git checkout -b feature/your-feature-name
# OR for bug fixes:
git checkout -b fix/bug-description
```

### 3. Make Your Changes & Commit
Keep your commits clean, focused, and descriptive.
```bash
git add .
git commit -m "Type a clear, imperative commit message (e.g., Fix login validation bug)"
```

### 4. Push and Submit a Pull Request
1. Push your branch up to your personal GitHub fork:
   ```bash
   git push origin feature/your-feature-name
   ```
2. Navigate back to the original repository on GitHub.
3. You will see a banner prompting you to open a **Pull Request**. Click it!

---

## 🛠️ Code Review & Protection Rules

To maintain high code quality, our repository enforces strict branch protection rules. When you submit a Pull Request, please note:

* **No Direct Pushes**: All code changes must go through a Pull Request. Direct pushes to `main` are automatically blocked.
* **Code Review Required**: At least one maintainer must review and approve your code before it can be merged.
* **Conversation Resolution**: If a reviewer leaves a comment or question on your code, the thread must be marked as "Resolved" before the merge button unlocks.
* **No Stale Code**: If you push new commits to your PR *after* it has been approved, the previous approval is automatically revoked, and a new review will be required.

## 🤝 Community Standards

By participating in this project, you agree to abide by our **Code of Conduct**. Please remain respectful, helpful, and welcoming to all other contributors.
